Settings

UI & Editor

Choose which editor Kestrel should use when opening originals. “System Default” will use your OS default application.
Full path to an application executable (e.g. C:\Program Files\MyEditor\editor.exe or /Applications/MyEditor.app).
How many subfolder levels to scan when building the folder tree (1–6). Higher values may increase scan time.

Star Rating System

Controls how quality scores are converted to star ratings. Each profile uses fixed quality-score thresholds — stricter profiles require a higher score to earn 4 or 5 stars. Changes take effect immediately for loaded folders.

Analysis Settings

Minimum confidence score (0.10–0.99) that the bird detector must reach before Kestrel counts something as a detection. Lowering this value may increase false positive detections on difficult images. Requires re-analysis to take effect. Also filters which species and family tags are shown in scene dialogs.
Images captured within this many seconds of each other are treated as the same burst/scene (using EXIF timestamp). Requires re-analysis to take effect.
Confidence threshold for mask segmentation (0.50–0.95). Higher values produce tighter masks around birds, lower values are more inclusive. Requires re-analysis to take effect.
Controls histogram-constrained exposure correction during analysis. Aggressive pushes brightness as far as possible without violating clipping/highlight guardrails; lenient and normal use progressively softer lift and darkening. Requires re-analysis to take effect.

Data & Preview

Cache full-resolution RAW previews in .kestrel/culling_TMP for faster subsequent loads. Disable to save disk space (previews will be generated fresh each time).
By default, Kestrel applies a calculated exposure boost to RAW previews to match the processed JPEG. Enable this to view the RAW data without correction.
Automatically save ratings, tags, and scene changes. When disabled, you'll see a Save/Revert workflow.

💬 Send Feedback

📊 Help Improve Project Kestrel

Would you like to share anonymous usage data to help improve the app? This is completely optional and you can change your mind at any time.

What would be shared (anonymously):

  • Number and size of images analyzed
  • File formats used (e.g. CR3, JPG)
  • Analysis duration
  • App version and OS type

What is never shared:

  • Filenames, folder paths, or image content
  • Any personally identifiable information

Your machine is identified only by a random ID — no device name, username, or IP address is stored or sent.
